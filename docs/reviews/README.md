# QuantaEngine 代码库评估档案

本目录保存按提交基线冻结的代码审查与架构评估。评估报告只记录当时可验证的事实，不覆盖旧报告；修复后新增复审记录，并通过 finding ID 关联原问题。

## 状态约定

| 状态 | 含义 |
|---|---|
| `open` | 已确认，尚未修复 |
| `planned` | 已有明确修复计划 |
| `fixed` | 已实现，但尚未完成独立复验 |
| `verified` | 修复和回归证据均已通过 |
| `accepted-risk` | 明确接受风险并记录原因 |

## 严重度约定

| 等级 | 含义 |
|---|---|
| `P0` | 数据破坏、安全事故或核心系统不可用，需要立即停止发布 |
| `P1` | 核心行为错误、结果不可复现或兼容性破坏，应优先修复 |
| `P2` | 可维护性、性能、测试和发布质量问题，应进入近期迭代 |
| `P3` | 局部改进或文档优化 |

## 复审流程

1. 从 `index.yaml` 选择仍为 `open` 或 `fixed` 的 finding。
2. 在修复提交中引用 finding ID，例如 `Fix QE-2026-001`。
3. 运行原报告中的复现步骤和验收标准。
4. 新建复审报告，不直接改写原始证据。
5. 将状态更新为 `verified`，记录修复提交和回归命令。
